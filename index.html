Mocha is a Javascript testing framework used for testing apps automatically.

INSTALLATION

Install with npm globally


						$ npm install --global mocha

or as a development dependency


						$ npm install --save-dev mocha

To check the installation, use the command


						$ mocha --version

describe('tittle', function() { ... }) The function takes two arguments: a String with tittle and a callback.

it('description', function() { ... }) Also takes two arguments. A string that contains a description of what the test should do and a callback function that contains the test itself.

ARROW FUNCTIONS


						describe('my suite', () => {
              it('my test', () => {
                // should set the timeout of this test to 1000 ms; instead will fail
                this.timeout(1000);

PENDING TESTS, EXCLUSIVE TESTS, INCLUSIVE TESTS


            describe('Array', function() {
              describe.only('#indexOf()', function() {
                it.only('should return -1 unless present', function() {
                  // this test will be run
                });
                it.skip('should return -1 unless present', function() {
                  // this test will not be run
                });
                it('should return the index when present', function() {
                  // this test will not be run
                });
                it('should return -1 the index when present');
                 // this test will not be run
            });

INTERFACES

  • BDD

    The BDD interface provides describe(), context(), it(), specify(), before(), after(), beforeEach(), and afterEach(). All of the previous examples were written using the BDD interface.

  • ТDD

    The TDD interface provides suite(), test(), suiteSetup(), suiteTeardown(), setup(), and teardown():

  • EXPORTS

    The Exports interface is much like Mocha’s predecessor expresso. The keys before, after, beforeEach, and afterEach are special-cased, object values are suites, and function values are test-cases:

  • QUNIT

    Like TDD, it uses suite() and test(), but resembling BDD, it also contains before(), after(), beforeEach(), and afterEach().
